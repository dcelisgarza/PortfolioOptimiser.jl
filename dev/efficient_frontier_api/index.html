Structure for a mean-variance portfolio.

  • tickers: list of tickers.
  • mean_ret: mean returns, don't need it to optimise for minimum variance.
  • weights: weight of each ticker in the portfolio.
  • cov_mtx: covariance matrix.
  • rf: risk free rate.
  • market_neutral: whether a portfolio is market neutral or not. Used in max_quadratic_utility!, efficient_risk!, efficient_return!.
  • risk_aversion: risk aversion parameter. Used in max_quadratic_utility!.
  • target_volatility: target volatility parameter. Used in efficient_risk!.
  • target_ret: target return parameter. Used in efficient_return!.
  • extra_vars: extra variables for the model.
  • extra_constraints: extra constraints for the model.
  • extra_obj_terms: extra objective terms for the model.
  • model: model for optimising portfolio.

PortfolioOptimiser.MeanVar — Method
MeanVar(
    tickers,
    mean_ret,
    cov_mtx;
    weight_bounds = (0, 1),
    rf = 0.02,
    market_neutral = false,
    risk_aversion = 1.0,
    target_volatility = rank(cov_mtx) < size(cov_mtx, 1) ? 1 / sum(diag(cov_mtx)) :
                        sqrt(1 / sum(inv(cov_mtx))),
    target_ret = mean(mean_ret),
    extra_vars = [],
    extra_constraints = [],
    extra_obj_terms = [],
)

Create an MeanVar structure to be optimised via JuMP.

  • tickers: list of tickers.
  • mean_ret: mean returns, don't need it to optimise for minimum variance.
  • cov_mtx: covariance matrix.
  • weight_bounds: weight bounds for tickers. If it's a Tuple of length 2, the first entry will be the lower bound for all weights, the second entry will be the upper bound for all weights. If it's a vector, its length must be equal to that of tickers, each element must be a tuple of length 2. In that case, each tuple corresponds to the lower and upper bounds for the corresponding ticker. See _create_weight_bounds for further details.
  • rf: risk free rate. Must be consistent with the frequency at which mean_ret and cov_mtx were calculated. The default value assumes daily returns.
  • market_neutral: whether a portfolio is market neutral or not. If it is market neutral, the sum of the weights will be equal to 0, else the sum will be equal to 1. Used in max_quadratic_utility!, efficient_risk!, efficient_return!.
  • risk_aversion: risk aversion parameter, the larger it is, the lower the risk. Used in max_quadratic_utility!.
  • target_volatility: target volatility parameter. Used in efficient_risk!.
  • target_ret: target return parameter. Used in efficient_return!.
  • extra_vars: extra variables for the model. See _add_var_to_model! for details on how to use this.
  • extra_constraints: extra constraints for the model. See _add_constraint_to_model! for details on how to use this.
  • extra_obj_terms: extra objective terms for the model. See _add_to_objective! for details on how to use this.

PortfolioOptimiser.max_return — Method
max_return(portfolio::MeanVar; optimiser = Ipopt.Optimizer, silent = true, optimiser_attributes = ())

Maximise the return (port_return) of a MeanVar portfolio. Internally minimises the negative of the portfolio return.

  • portfolio: MeanVar structure.
  • optimiser: JuMP-supported optimiser, must support quadratic objectives.
  • silent: if true the optimiser will not print to console, if false the optimiser will print to console.
Warning

This should not be used for optimising portfolios. It's used by efficient_return! to validate the target return. This yields portfolios with large volatilities.

PortfolioOptimiser.max_sharpe! — Function
max_sharpe!(
    portfolio::MeanVar,
    rf = portfolio.rf;
    optimiser = Ipopt.Optimizer,
    silent = true,
    optimiser_attributes = (),
)

Maximise the sharpe ratio (sharpe_ratio) of a MeanVar portfolio.

Uses a variable transformation to turn the nonlinear objective that is the sharpe ratio, into a convex quadratic minimisation problem. See Cornuejols and Tutuncu (2006) page 158 for more details.

  • portfolio: MeanVar structure.
  • rf: risk free rate.
  • optimiser: JuMP-supported optimiser, must support quadratic objectives.
  • silent: if true the optimiser will not print to console, if false the optimiser will print to console.
Warning

The variable transformation also modifies every constraint in portfolio, including all extra constraints. Therefore, one should not call other optimsiations on the same instance after optimising the sharpe ratio, create a fresh intance instead.

Warning

The variable transformation means any extra terms in the objective function may not work as intended. If you need to add extra objective terms, use custom_nloptimiser! (see the example) and add the extra objective terms in the objective function.
