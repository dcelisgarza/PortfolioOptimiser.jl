Creates an instance of Portfolio containing all internal data necessary for convex portfolio optimisations as well as failed and successful results.

Inputs

Portfolio characteristics

  • prices: (T+1)×Na TimeArray with asset pricing information, where the time stamp field is timestamp, where T is the number of returns observations and Na is the number of assets. If prices is not empty, then returns, ret, timestamps, assets, and latest_prices are ignored because their respective fields are obtained from prices.
  • returns: T×(Na+1) DataFrame, where T is the number of returns observations and Na is the number of assets, the extra column is timestamp, which contains the timestamps of the returns. If prices is empty and returns is not empty, ret, timestamps, and assets are ignored because their respective fields are obtained from returns.
  • ret: T×Na matrix of returns, where T is the number of returns observations and Na is the number of assets. Its value is saved in the returns field of Portfolio. If prices or returns are not empty, this value is obtained from within the function, where T is the number of returns observations and Na is the number of assets.
  • timestamps: T×1 vector of timestamps, where T is the number of returns observations. Its value is saved in the timestamps field of Portfolio. If prices or returns are not empty, this value is obtained from within the function.
  • assets: Na×1 vector of assets, where Na is the number of assets. Its value is saved in the assets field of Portfolio. If prices or returns are not empty, this value is obtained from within the function.
  • short: whether or not to allow negative weights, i.e. whether a portfolio accepts shorting.
  • short_u: absolute value of the sum of all short (negative) weights.
  • long_u: sum of all long (positive) weights.
  • min_number_effective_assets: if non-zero, guarantees that at least number of assets make significant contributions to the final portfolio weights.
  • max_number_assets: if non-zero, guarantees at most this number of assets make non-zero contributions to the final portfolio weights. Requires an optimiser that supports binary variables.
  • max_number_assets_factor: scaling factor needed to create a decision variable when max_number_assets is non-zero.
  • f_prices: (T+1)×Nf TimeArray with factor pricing information, where the time stamp field is f_timestamp, where T is the number of returns observations and Nf is the number of factors. If f_prices is not empty, then f_returns, f_ret, f_timestamps, f_assets, and latest_prices are ignored because their respective fields are obtained from f_prices.
  • f_returns: T×(Nf+1) DataFrame, where T is the number of returns observations and Nf is the number of factors, the extra column is f_timestamp, which contains the timestamps of the factor returns. If f_prices is empty and f_returns is not empty, f_ret, f_timestamps, and f_assets are ignored because their respective fields are obtained from f_returns.
  • f_ret: T×Nf matrix of factor returns, where Nf is the number of factors. Its value is saved in the f_returns field of Portfolio. If f_prices or f_returns are not empty, this value is obtained from within the function, where T is the number of returns observations and Nf is the number of factors.
  • f_timestamps: T×1 vector of factor timestamps, where T is the number of returns observations. Its value is saved in the f_timestamps field of Portfolio. If f_prices or f_returns are not empty, this value is obtained from within the function.
  • f_assets: Nf×1 vector of assets, where Nf is the number of factors. Its value is saved in the f_assets field of Portfolio. If f_prices or f_returns are not empty, this value is obtained from within the function.
  • loadings: loadings matrix for black litterman models.

Risk parameters

  • msv_target: target value for for Absolute Deviation and Semivariance risk measures. It can have two meanings depending on its type and value.
    • If it's a Real number and infinite, or an empty vector. The target will be the mean returns vector mu.
    • Else the target is the value of msv_target. If msv_target is a vector, its length should be Na, where Na is the number of assets.
  • lpm_target: target value for the First and Second Lower Partial Moment risk measures. It can have two meanings depending on its type and value.
    • If it's a Real number and infinite, or an empty vector. The target will be the value of the risk free rate rf, provided in the opt_port! function.
    • Else the target is the value of lpm_target. If lpm_target is a vector, its length should be Na, where Na is the number of assets.
  • alpha_i: initial significance level of Tail Gini losses, 0 < alpha_i < alpha < 1.
  • a_sim: number of CVaRs to approximate the Tail Gini losses, a_sim > 0.
  • alpha: significance level of VaR, CVaR, EVaR, RVaR, DaR, CDaR, EDaR, RDaR, CVaR losses, or Tail Gini losses, depending on the RiskMeasures and upper bounds being used, alpha ∈ (0, 1).
  • beta_i: initial significance level of Tail Gini gains, 0 < beta_i < beta < 1.
  • b_sim: number of CVaRs to approximate the Tail Gini gains, b_sim > 0.
  • beta: significance level of CVaR gains or Tail Gini gains, depending on the RiskMeasures and upper bounds being used, beta ∈ (0, 1).
  • kappa: deformation parameter for relativistic risk measures (RVaR and RDaR).
  • gs_threshold: Gerber statistic threshold.
  • max_num_assets_kurt: maximum number of assets to use the full kurtosis model, if the number of assets surpases this value use the relaxed kurtosis model.

Benchmark constraints

  • turnover: if finite, define the maximum turnover deviations from turnover_weights to the optimised portfolio. Else the constraint is disabled.
  • turnover_weights: target weights for turnover constraint.
    • The turnover constraint is defined as $\lvert w_{i} - \hat{w}_{i}\rvert \leq e_{1} \, \forall\, i \in N$, where $w_i$ is the optimal weight for the i'th asset, $\hat{w}_i$ target weight for the i'th asset, $e_{1}$ is the value of the turnover, and $N$ is the number of assets.
  • kind_tracking_err: :Weights when providing a vector of asset weights for computing the tracking error benchmark from the asset returns, or :Returns to directly providing the tracking benchmark. See TrackingErrKinds for more information.
  • tracking_err: if finite, define the maximum tracking error deviation. Else the constraint is disabled.
  • tracking_err_returns: T×1 vector of returns to be tracked, where T is the number of returns observations. When kind_tracking_err == :Returns, this is used directly tracking benchmark.
  • tracking_err_weights: Na×1 vector of weights, where Na is the number of assets, when kind_tracking_err == :Weights, the returns benchmark is computed from the returns field of Portfolio.
    • The tracking error is defined as $\sqrt{\dfrac{1}{T-1}\sum\limits_{i=1}^{T}\left(\mathbf{X}_{i} \bm{w} - b_{i}\right)^{2}}\leq e_{2}$, where $\mathbf{X}_{i}$ is the i'th observation (row) of the returns matrix $\mathbf{X}$, $\bm{w}$ is the vector of optimal asset weights, $b_{i}$ is the i'th observation of the benchmark returns vector, $e_{2}$ the tracking error, and $T$ is the number of returns observations.
  • bl_bench_weights: Na×1 vector of benchmark weights for Black Litterman models, where Na is the number of assets.

Risk and return constraints

  • a_mtx_ineq: C×Na A matrix of the linear asset constraints $\mathbf{A} \bm{w} \geq \bm{B}$, where C is the number of constraints, and Na is the number of assets.
  • b_vec_ineq: C×1 B vector of the linear asset constraints $\mathbf{A} \bm{w} \geq \bm{B}$, where C is the number of constraints.
  • risk_budget: Na×1 risk budget constraint vector for risk parity optimisations, where Na is the number of assets.

Bounds constraints

The bounds constraints are only active if they are finite. They define lower bounds denoted by the suffix _l, and upper bounds denoted by the suffix _u, of various portfolio characteristics. The risk upper bounds are named after their corresponding RiskMeasures in lower case. Multiple bounds constraints can be active at any time but may make finding a solution infeasable.

  • mu_l: mean expected return.
  • dev_u: standard deviation.
  • mad_u: max absolute devia.
  • sdev_u: semi standard deviation.
  • cvar_u: critical value at risk.
  • wr_u: worst realisation.
  • flpm_u: first lower partial moment.
  • slpm_u: second lower partial moment.
  • mdd_u: max drawdown.
  • add_u: average drawdown.
  • cdar_u: critical drawdown at risk.
  • uci_u: ulcer index.
  • evar_u: entropic value at risk.
  • edar_u: entropic drawdown at risk.
  • rvar_u: relativistic value at risk.
  • rdar_u: relativistic drawdown at risk.
  • kurt_u: square root kurtosis.
  • skurt_u: square root semi kurtosis.
  • gmd_u: gini mean difference.
  • rg_u: range.
  • rcvar_u: critical value at risk range.
  • tg_u: tail gini.
  • rtg_u: tail gini range.
  • owa_u: custom ordered weight risk (use with owa_w).

Custom OWA weights

  • owa_w: T×1 OWA vector, where T is the number of returns observations containing. Useful for optimising higher OWA L-moments.

Model statistics

  • mu_type: method for estimating the mean returns vectors mu, mu_fm, mu_bl, mu_bl_fm in mean_vec, see MuTypes for available choices.
  • mu: Na×1 vector, where Na is the number of assets). Set the value of the asset mean returns at instance construction. When choosing :Custom_Val in mu_type, this is the value of mu used, can also be set after a call to mean_vec to replace the old value with the new. Can be directly computed by asset_statistics!.
  • cov_type: methods for estimating the covariance matrices cov, cov_fm, cov_bl, cov_bl_fm in covar_mtx, see CovTypes for available choices.
  • jlogo: if true, apply the j-LoGo transformation to the portfolio covariance matrix in covar_mtx [jLoGo].
  • cov: Na×Na matrix, where Na is the number of assets. Set the value of the asset covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by asset_statistics!.
  • kurt: (Na×Na)×(Na×Na) matrix, where Na is the number of assets. Set the cokurtosis matrix at instance construction. The cokurtosis matrix kurt can be computed by calling cokurt_mtx.
  • skurt: (Na×Na)×(Na×Na) matrix, where Na is the number of assets. Set the semi cokurtosis matrix at instance construction. The semi cokurtosis matrix skurt can be computed by calling cokurt_mtx.
  • posdef_fix: method for fixing non positive definite matrices when computing portfolio statistics, see PosdefFixes for available choices.
  • mu_f: Nf×1 vector, where Nf is the number of factors). Set the value of the factors mean returns at instance construction. When choosing :Custom_Val in mu_type, this is the value of mu used, can also be set after a call to mean_vec to replace the old value with the new. Can be directly computed by factor_statistics!.
  • cov_f: Nf×Nf matrix, where Nf is the number of factors. Set the value of the factors covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by factor_statistics!.
  • mu_fm: Na×1 vector, where Na is the number of assets). Set the value of the feature selected factors mean returns at instance construction. When choosing :Custom_Val in mu_type, this is the value of mu used, can also be set after a call to mean_vec to replace the old value with the new. Can be directly computed by factor_statistics!.
  • cov_fm: Na×Na matrix, where Na is the number of assets. Set the value of the feature selected factors covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by factor_statistics!.
  • mu_bl: Na×1 vector, where Na is the number of assets). Set the value of the Black Litterman mean returns at instance construction. When choosing :Custom_Val in mu_type, this is the value of mu used, can also be set after a call to mean_vec to replace the old value with the new. Can be directly computed by black_litterman_statistics!.
  • cov_bl: Na×Na matrix, where Na is the number of assets. Set the value of the Black Litterman covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by black_litterman_statistics!.
  • mu_bl_fm: Na×1 vector, where Na is the number of assets). Set the value of the Black Litterman feature selected factors mean returns at instance construction. When choosing :Custom_Val in mu_type, this is the value of mu used, can also be set after a call to mean_vec to replace the old value with the new. Can be directly computed by black_litterman_factor_satistics!.
  • cov_bl_fm: Na×Na matrix, where Na is the number of assets. Set the value of the Black Litterman feature selected factors covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by black_litterman_factor_satistics!.
  • returns_fm: T×Na matrix of feature selcted adjusted returns, where T is the number of returns observations and Na is the number of assets. Can be directly computed by factor_statistics!.

Inputs of Worst Case Optimization Models

  • cov_l: Na×Na matrix, where Na is the number of assets. Set the value of the worst case lower bound asset covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by wc_statistics!.
  • cov_u: Na×Na matrix, where Na is the number of assets. Set the value of the worst case upper bound asset covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by wc_statistics!.
  • cov_mu: Na×Na matrix, where Na is the number of assets. Set the value of the estimation errors of the mean vector covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by wc_statistics!.
  • cov_sigma: (Na×Na)×(Na×Na) matrix, where Na is the number of assets. Set the value of the estimation errors of the covariance matrix covariance matrix at instance construction. When choosing :Custom_Val in cov_type, this is the value of cov used by covar_mtx. Can be directly computed by wc_statistics!.
  • d_mu: Na×1 vector, where Na is the number of assets). Set the value of the delta mean returns at instance construction. When choosing :Custom_Val in mu_type, this is the value of mu used, can also be set after a call to mean_vec to replace the old value with the new. Can be directly computed by wc_statistics!.
  • k_mu: set the percentile of a sample of size Na, where Na is the number of assets, at instance creation. Can be directly computed by wc_statistics!.
  • k_sigma: set the percentile of a sample of size Na×Na, where Na is the number of assets, at instance creation. Can be directly computed by wc_statistics!.

Optimal portfolios

  • optimal: Empty concrete subtype of AbstractDictionary for storing optimal portfolios. This parameter is filled after calling opt_port!.
  • z: Empty concrete subtype of AbstractDictionary for storing optimal z values of portfolios optimised for entropy and relativistic risk measures. This parameter is filled after calling opt_port!.
  • limits: Empty concrete subtype of AbstractDictionary for storing the minimal and maximal risk portfolios for given risk measures. This parameter is filled after calling frontier_limits!.
  • frontier: Empty concrete subtype of AbstractDictionary containing points in the efficient frontier for given risk measures. This parameter is filled after calling efficient_frontier!.

Solutions

  • solvers: Provides the solvers and corresponding parameters for solving risk measure JuMP model. Dict or NamedTuple with key value pairs where the values are other Dicts or NamedTuples, e.g. Dict(solver_key => Dict(...)), the keys of the sub-dictionary/tuple must be:
    • :solver: which contains the JuMP optimiser.
    • :params: (optional) for the solver-specific parameters.
Note

In order for the optimisation to succeed, a solver needs to support a variety of cones and constraints depending on the risk measure, bounds, and constraints. Alternatively, JuMP must be able to transform it/them, into a supported form.

  • opt_params: Empty concrete subtype of AbstractDictionary for storing parameters used for optimising. This parameter is filled after calling opt_port!.
  • fail: Empty concrete subtype of AbstractDictionary for storing failed optimisation attempts. This parameter is filled after calling opt_port!.
  • model: JuMP.Model() for optimising a portfolio. This parameter is filled after calling opt_port!.

Allocation

  • latest_prices: Na×1 vector of asset prices, Na is the number of assets. If prices is not empty, this is automatically obtained from the last entry. This is used for discretely allocating stocks according to their prices, weight in the portfolio, and money to be invested.
  • alloc_optimal: Empty concrete subtype of AbstractDictionary for storing optimal portfolios after allocating discrete stocks. This parameter is filled after calling allocate_port!.
  • alloc_solvers: Provides the solvers and corresponding parameters for solving discrete allocation JuMP model. Dict or NamedTuple with key value pairs where the values are other Dicts or NamedTuples, e.g. Dict(solver_key => Dict(...)), the keys of the sub-dictionary/tuple must be:
    • :solver: which contains the JuMP optimiser.
    • :params: (optional) for the solver-specific parameters.
Note

In order for the optimisation to succeed, a solver needs to support mixed-integer problems. Alternatively, JuMP must be able to transform it/them, into a supported form.

  • alloc_params: Empty concrete subtype of AbstractDictionary for storing parameters used for optimising the portfolio allocation. This parameter is filled after calling allocate_port!.
  • alloc_fail: Empty concrete subtype of AbstractDictionary for storing failed optimisation attempts. This parameter is filled after calling allocate_port!.
  • alloc_model: JuMP.Model() for optimising a portfolio allocation. This parameter is filled after calling allocate_port!.

PortfolioOptimiser.TrackingErrKinds — Constant
TrackingErrKinds = (:Weights, :Returns)

Available kinds of tracking errors for Portfolio.

  • :Weights: provide a vector of asset weights which is used to compute the vector of benchmark returns,
    • $\bm{b} = \mathbf{X} \bm{w}$,

where $\bm{b}$ is the benchmark returns vector, $\mathbf{X}$ the $T \times{} N$ asset returns matrix, and $\bm{w}$ the asset weights vector.

  • :Returns: directly provide the vector of benchmark returns.

The benchmark is then used as a reference to optimise a portfolio that tracks it up to a given error.
